# 智能战报数据层改造方案

## 1. 现状分析

### 1.1 数据维度

| 维度 | 层级 | 数量 |
|------|------|------|
| 区域 | 地区部 → 代表处 → 国家/地区 | 5 → 22 → 125 |
| 产品 | 产品类别 → 产品系列 → 产品 | 6 → 35 → 179 |
| 指标 | PSI Type | 3 (Sell In / Sell Out / Inventory) |

### 1.2 现有数据格式问题

当前样例数据采用**宽表格式**（日期作为列），存在以下问题：

1. **列爆炸**：按日展开后，561天的数据会产生561个日期列，维护困难
2. **稀疏存储**：大量零值或无数据组合仍占用存储空间
3. **查询效率低**：时间范围查询需要扫描大量列
4. **扩展性差**：新增日期需要修改表结构

---

## 2. 改造方案

### 2.1 核心设计原则

| 原则 | 说明 |
|------|------|
| 长表存储 | 日期从"列"转为"行"，降低列数 |
| 分层存储 | 原始数据(天) + 聚合数据(周/月/季) |
| 稀疏处理 | 无销量组合不存储，降低存储量 |
| 预计算 | 常用聚合指标提前计算，加速查询 |
| 版本管理 | 数据变更可追溯，支持数据回溯 |

### 2.2 数据模型设计

#### 2.2.1 维度表（星型模型）

```
┌─────────────────────────────────────────────────────────────────────┐
│                        事实表: psi_daily                             │

│  date_key │ region_key │ product_key │ psi_type │ value │ data_version │

      ▼            ▼            ▼           ▼         ▼          ▼

│ dim_date │  │ dim_region│  │ dim_product│          │dim_version│

```

### 2.3 存储格式对比

| 格式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| Excel (.xlsx) | 易读、易编辑 | 大数据量性能差 | 小样本、人工查看 |
| CSV | 通用、易生成 | 无压缩、类型丢失 | 数据交换 |
| Parquet | 列式存储、压缩率高 | 需要专用工具读取 | 大数据存储、分析 |
| SQLite | 轻量、SQL支持 | 并发写入受限 | 本地分析、原型 |
| DuckDB | 分析型、性能高 | 嵌入式 | 本地OLAP分析 |

**推荐方案**：
- **开发测试**：SQLite / DuckDB（轻量、SQL支持）
- **生产存储**：Parquet（列式存储，适合时间序列分析）
- **数据交换**：CSV（通用兼容）

### 2.4 数据生成规则（业务逻辑）

#### 2.4.1 稀疏性规则

并非所有 (国家, 产品) 组合都有销量，需引入**市场渗透率**概念：

```
渗透率 = f(国家经济发展水平, 产品定位, 历史销量)
```

- **旗舰产品**（如P系列手机）：在所有国家均有销售
- **入门级产品**（如nova Y系列）：在新兴市场渗透率更高
- **车载/信创产品**：仅在特定国家销售
- **随机因素**：10%~30% 的组合无销量

#### 2.4.2 数值生成规则

**Sell In（进货）**：
- 基础量：由产品定位决定（旗舰 > 精品 > 入门）
- 季节性：Q4 > Q2 > Q3 > Q1（年底备货高峰）
- 趋势：整体逐年微增（2%~5%）
- 噪声：±20% 随机波动

**Sell Out（销售）**：
- 与 Sell In 强相关，但有滞后性
- 周末效应：周末销量下降 20%~40%
- 促销节点：双11、黑五、圣诞等销量激增 2~3 倍
- 季节性：Q4 > Q2 > Q3 > Q1

**Inventory（库存）**：
- 逻辑：Inventory(t) = Inventory(t-1) + SellIn(t) - SellOut(t)
- 约束：Inventory ≥ 0（不允许负库存）
- 初始库存：基于前30天平均 Sell In × 安全系数(1.5~3)
- 目标：库存周转天数维持在 15~45 天

#### 2.4.3 数据校验规则

| 校验项 | 规则 |
|--------|------|
| 非负性 | Sell In ≥ 0, Sell Out ≥ 0, Inventory ≥ 0 |
| 库存平衡 | Inventory(t) = Inventory(t-1) + SellIn(t) - SellOut(t) |
| 合理范围 | 单日单产品销量不超过 10,000 |
| 比例关系 | Sell Out / Sell In 比值在 0.5~2.0 之间 |

## 3. 实施方案

### 3.1 阶段划分

| 阶段 | 任务 | 产出 |
|------|------|------|
| Phase 1 | 维度表建设 | dim_region, dim_product, dim_date |
| Phase 2 | 事实表生成 | psi_daily（日级数据） |
| Phase 3 | 聚合表建设 | psi_weekly, psi_monthly, psi_quarterly |
| Phase 4 | 数据校验 | 规则引擎、质量报告 |
| Phase 5 | 接口层 | API / SQL 视图封装 |

### 3.2 文件组织

```
数据层/
├── raw/                    # 原始数据（宽表格式，兼容旧系统）
│   └── psi_raw_YYYYMMDD.xlsx
├── dimensions/             # 维度表
│   ├── dim_region.parquet
│   ├── dim_product.parquet
│   └── dim_date.parquet
├── facts/                  # 事实表
│   ├── psi_daily.parquet   # 日级明细
│   ├── psi_weekly.parquet  # 周级聚合
│   ├── psi_monthly.parquet # 月级聚合
│   └── psi_quarterly.parquet # 季度聚合
├── exports/                # 导出文件
│   └── psi_export_YYYYMMDD.csv
└── metadata/               # 元数据
    └── data_dictionary.md
```

## 4. 样例数据说明

本次生成的样例数据范围：**2025-01-01 至 2026-07-14**（共561天）

### 4.1 数据特征

- **稀疏性**：约 60% 的 (国家, 产品) 组合有实际销量数据
- **季节性**：Q4 销量显著高于其他季度
- **周末效应**：周末 Sell Out 下降 30%
- **库存联动**：Inventory 严格遵循 期初+进货-销售 的公式
- **数据格式**：采用宽表格式输出（兼容现有系统），同时生成长表格式

### 4.2 输出文件

| 文件 | 格式 | 说明 |
|------|------|------|
| psi_data_wide.xlsx | Excel | 宽表格式，兼容现有系统 |
| psi_data_long.csv | CSV | 长表格式，便于分析 |
| data_summary.md | Markdown | 数据摘要报告 |
