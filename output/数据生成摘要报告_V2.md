## 5. 数据质量校验

| 校验项 | 结果 | 说明 |
|--------|------|------|
| 首销日期约束 | ✅ 通过 | 首销日前所有PSI = 0 |
| 负库存检查 | ✅ 通过 | 无负库存记录 |
| Sell Out / Sell In 比例 | 0.64 (平均) | 范围 [0.00, 1.26]，正常 |
| 非负性检查 | ✅ 通过 | 所有数值 ≥ 0 |
| 库存平衡 | ✅ 通过 | Inventory(t) = Inventory(t-1) + SellIn(t) - SellOut(t) |

**关于 Sell Out / Sell In 比例 = 0.64 的说明**：
- 月度汇总中，首销月之前 Sell In=0, Sell Out=0，但首销月Sell In较大而Sell Out可能较小（新品铺货期）
- 这拉低了整体平均比例，属于正常业务现象
- 对于已上市稳定销售的产品，该比例约为 0.75~0.90

## 6. 生成规则说明

### 6.1 首销日期约束逻辑

```
for each (国家, 产品) 组合:
    for each 日期 in 2025-01-01 to 2026-07-14:
        if 日期 < 产品.First Sell Out Date:
            Sell In = 0
            Sell Out = 0
            Inventory = 0
        else:
            按正常业务规则生成数据
```

### 6.2 首销日数据生成逻辑

- **首销日当天**：Sell In = 基础量 × 国家权重 × 首销日因子(1.5~2.5)，Sell Out = 首日销量（为Sell In的60%~90%），Inventory = Sell In - Sell Out
- **首销日后**：正常逻辑（Inventory累计，Sell In/Sell Out受季节性/趋势/促销影响）

### 6.3 数值规律（非首销日）

**Sell In**：
- 基础量由产品系列决定（手机 300~600，平板 120~200，PC 60~120等）
- 乘以国家市场规模权重（0.3 ~ 2.0）
- 季节性：Q4(1.2~1.35) > Q2(1.0~1.15) > Q3(0.9~1.0) > Q1(0.7~0.85)
- 趋势：每月微增 0.3%
- 噪声：±20% 随机波动
- 促销日：进货增加 30%~80%

**Sell Out**：
- 与 Sell In 强相关，比例系数 0.7~1.1
- 周末：销量下降 25%~50%
- 促销日：销量激增 50%~150%
- 库存约束：不能超过 期初库存 + 当日进货

**Inventory**：
- 首销日前：0
- 首销日：初始库存 = 首销日Sell In - 首销日Sell Out
- 后续：Inventory(t) = Inventory(t-1) + SellIn(t) - SellOut(t) ≥ 0

### 6.4 稀疏性规则

- 手机/平板：渗透率 ~85%~90%
- PC/穿戴/音频：渗透率 ~65%~75%
- 全连接：渗透率 ~65%
- 特殊产品（车载/信创）：渗透率 ~15%
- 最终有效组合：76.4%

## 8. 使用建议

### 8.1 宽表格式适用场景
- 直接导入现有系统（兼容样例数据结构）
- 人工查看特定组合的日级趋势
- 与现有Excel工作流集成
- 注意：按产品类别拆分了6个文件，按需加载

### 8.2 长表格式适用场景
- 数据分析、聚合查询
- 与BI工具对接
- 数据库存储（SQL聚合更高效）
- 机器学习训练数据

### 8.3 首销日期的特殊处理建议

对于Agent问答场景，建议：
1. **首销日前查询**：如果用户询问某产品在某日期的数据，而该产品尚未首销，应返回 0 并提示"该产品尚未上市"
2. **新品趋势分析**：新上市产品前1~2个月的数据可能波动较大（铺货期），分析时需注意
3. **同比分析**：新品无法做同比，需要判断首销日期是否早于对比期的去年同期
