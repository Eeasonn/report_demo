## 一、项目概述

### 1.1 背景与目标

业界现有的 AI 问数、AI 做报告方案普遍存在不确定性高、语义配置复杂、用户后操作多等问题。本项目旨在构建一个**轻量化的战报智能体**，通过预设标准模板 + 有限 AI 修改的模式，在**确定性**与**智能化**之间取得平衡。

**核心目标**：
- 为用户提供自然语言入口，快速查找、查看战报
- 支持有限度的个性化修改（时间、重点机型）
- 支持订阅与分享，满足定时推送需求
- 后台承载所有语义配置，前台零配置

### 1.2 关键术语

| 术语 | 说明 |
|------|------|
| **战报库** | 标准战报的全集，按地区/产品维度组织 |
| **标准战报** | 由后台预定义的固定模板，修改需提需求 |
| **个性化战报** | 用户在标准战报基础上修改后保存的副本 |
| **PSI** | Product Sales Inventory，产品销存指标 |
| **SO** | Sell Out，销量（终端售出） |
| **Inv** | Inventory，库存 |
| **DOS** | Days of Supply，库存可销天数 |

### 3.2 战报库模块

#### 3.2.1 战报结构设计

```typescript
interface Report {
  id: string;                    // 唯一标识
  code: string;                  // 战报编码，如 "MEA_PHONE_DAILY"
  name: string;                  // 战报名称，如 "中东非洲手机日销战报"
  category: string;              // 分类：手机/耳机/穿戴/综合
  region: string;                // 地区：MEA/拉美/欧洲/亚太...
  type: 'daily' | 'weekly' | 'monthly';  // 周期类型
  template: ReportTemplate;      // 模板定义
  dataSource: string;            // 数据源标识
  permission: string[];          // 可见权限组
  defaultModels: string[];       // 默认重点机型
  description: string;           // 战报描述
}

interface ReportTemplate {
  sections: TemplateSection[];   // 模板段落
  style: 'series' | 'category' | 'comprehensive';  // 样式类型
}

interface TemplateSection {
  title: string;                 // 段落标题，如 "【折叠】"
  products: string[];            // 包含的产品系列
  showTotal: boolean;            // 是否显示合计
  showYoY: boolean;              // 是否显示同比
  showInventory: boolean;        // 是否显示库存
}
```

#### 3.3.3 确认与反问策略

**必须进行确认的场景**：
1. **模糊地区匹配**：用户说"欧洲"，实际有 3 个欧洲相关战报
   > "为您找到以下欧洲相关战报：\n1. 欧洲手机日销\n2. 欧洲综合周销\n3. 欧洲耳机日销\n\n请问您要查看哪一个？"

2. **修改操作**：用户要求修改任何参数
   > "确认将时间范围从 7.1-7.9 修改为 7.11-7.19 吗？"

3. **模糊机型匹配**："折叠机"匹配多个型号
   > "折叠机包含 Mate X7、Mate XT 等，您具体指哪一款？"

4. **保存个性化战报**：
   > "是否将修改后的战报保存为『我的欧洲手机日报（7.11-7.19版）』？"

#### 3.4.2 PSI 计算规则

| 指标 | 单日 | 时间区间（多日）|
|------|------|---------------|
| **SO (Sell Out)** | 取当日值 | **区间内求和** |
| **Inv (Inventory)** | 取当日值 | **取区间最后一天值** |
| **DOS** | Inv / 日均 SO | Inv / 区间日均 SO |

### 3.5 订阅与分享模块

#### 3.5.1 订阅管理

```typescript
interface Subscription {
  id: string;
  userId: string;
  reportId: string;           // 标准战报 ID
  personalizedConfig?: {       // 个性化配置（可选）
    timeRange?: string;
    focusModels?: string[];
  };
  schedule: {
    type: 'daily' | 'weekly' | 'monthly';
    time: string;             // 推送时间，如 "09:00"
    timezone: string;         // 时区
  };
  enabled: boolean;
  createdAt: Date;
}
```

**默认更新规则示例**：
- 日销战报：每日 09:00 推送，数据自动更新为昨日
- 周销战报：每周一 09:00 推送，数据自动更新为上周
- 所有战报时间参数自动计算，无需用户干预

## 六、权限与安全设计

### 6.1 权限模型

```
用户角色
    │
    ├── 超级管理员 (admin)
    │   └── 全部战报 + 系统配置
    │
    ├── 地区管理员 (region_admin)
    │   └── 所辖地区全部战报
    │
    ├── 普通用户 (user)
    │   └── 有权限的战报列表
    │
    └── 访客 (guest)
        └── 仅演示战报
```

### 6.2 数据隔离规则

1. **列表隔离**：用户只能看到 `permission` 数组包含其角色的战报
2. **搜索隔离**：AI 推荐时自动过滤无权限战报
3. **分享隔离**：分享链接可设置有效期和访问密码

## 十、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 意图识别准确率不足 | 高 | 采用 Few-shot Prompting + 意图分类器双层架构，关键操作强制确认 |
| 大模型响应延迟 | 中 | 意图识别本地缓存 + 战报数据预计算 + 流式输出 |
| Excel 数据量大时性能差 | 中 | Demo 阶段限制数据量，生产环境迁移至数据库 |
| 地区/产品名称变更 | 低 | 语义配置化，后台维护同义词映射 |
| 浏览器兼容性 | 低 | 基于 Ant Design X，默认兼容现代浏览器 |
